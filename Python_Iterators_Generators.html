# Python · Конспект

Итераторы,
итерируемые объекты
и генераторы

Три концепции, которые лежат в основе любого цикла в Python — и почему генераторы экономят память.

Iterable

Итерируемый объект

Объект, который предоставляет возможность обходить свои элементы по очереди. Имеет метод __iter__(), который возвращает итератор. Данные хранятся в памяти, обходить можно много раз.

list str tuple set dict range bytes
→ Любой итерируемый объект можно преобразовать в итератор с помощью iter().
Iterator

Итератор

Объект, который поддерживает функцию next() и помнит, какой элемент будет следующим. Хранит внутреннее состояние (текущую позицию). Движется только вперёд.

# создание итератора и обход
s = [1, 2, 3]
d = iter(s)          # создаём итератор

print(next(d))     # → 1
print(next(d))     # → 2
print(next(d))     # → 3
# next(d)  →  StopIteration  ← конец
вывод 
1
2
3
Generator

Генератор

Особый вид итератора. Элементы не хранятся в памяти — вычисляются и отдаются по одному при каждом next(). Пройтись по элементам можно только один раз.

# генераторное выражение
b = (i ** 2 for i in range(1, 6))

print(list(b))      # первый проход
print(list(b))      # второй проход → []
вывод 
[1, 4, 9, 16, 25]
[]
⚠ Нельзя применить индекс b[0] и узнать длину len(b). После исчерпания генератор пуст — второй list() вернёт пустой список.
# огромная последовательность — без нагрузки на память
# список — попытается хранить 10 млрд чисел ❌
s = [i for i in range(10_000_000_000)]

# генератор — хранит только текущую позицию ✅
c = (i for i in range(10_000_000_000))
yield

Функция-генератор

Функция с ключевым словом yield вместо return. При вызове возвращает объект-генератор. Выполнение «замораживается» на yield — все локальные переменные сохраняются.

★ yield означает «отдать» значение и приостановиться, сохранив всё состояние функции до следующего next().
1 Вызываем функцию → получаем генератор (код ещё не выполняется)
2 next() запускает функцию до первого yield → отдаёт значение
3 Функция «замораживается» — все локальные переменные сохранены в памяти
4 Следующий next() продолжает выполнение с того же места
# пример функции-генератора
def f(a):
    for i in range(7):
        yield i           # заморозка здесь

lis = f(10)               # генератор создан, код ещё не пошёл

for i in lis:
    print(i, end=' ')
вывод 
0 1 2 3 4 5 6
